Projektbeschreibung

Die Aufgabe war es in Einzelarbeit eine Grails Applikation mit den im Unterricht erlernten Mitteln zu erstellen und diese auf Github zu stellen. Der Aufwand sollte ungefähr 24 Stunden umfassen.

Der Student reicht eine Idee ein. Diese wird vom Dozenten via Github geprüft und genehmigt.

Projektidee

Umgesetzt wird eine Applikation die es einem Studenten ermöglicht seinen Studiumsfortschritt zu erfassen. Dazu kann er Semester erfassen und für jedes Semester die Module erfassen, welche er besucht. Ein Modul kann erfasst werden und für ein Modul kann definiert werden wie die Modulnote gebildet wird (Testatarbeit, Nur MSP, Anzahl Noten für EN, etc).

Dem Studenten wird in einem Dashboard sein aktuelles Semester, den Fortschritt und den gewichteten und ungewichteten Notendurchschnitt dargestellt.

Datenmodell

Das Datenmodell beschreibt die Relationen zwischen Modul, Semester und Note. Dabei ist darauf zu achten, dass ein Modul eine 0..1 Relation zu Note hat. Dies ist, falls das Modul eine MSP hat. Zudem gibt es eine 1..n Relation zwischen Modul und Note. Dies sind die Noten während des Semesters.

Beispieldaten

Um die Applikation mit Daten zu testen liegt ein sql-Skript mit Bespieldaten vor. Dieses ist in scripts/data.sql zu finden. Die Daten können nach dem Start der Applikation über die H2-Console importiert werden.

Per Default werden drei Benutzer angelegt:

  • Admin: admin/admin
  • User: user/user
  • Guest: guest/guest

Die Benutzer haben unterschiedliche Berechtigungen (ROLE_ADMIN, ROLE_STUDENT, ROLE_GUEST). Jede Rolle kann andere Inhalte sehen bzw. nicht anzeigen lassen.

Tests

Es wurden für alle Controller Tests geschrieben. Für die Services wurden ebenfalls Tests geschrieben. Zudem wurden für die Klasse Modul Unit Tests geschrieben.

Die Testabdeckung liegt jedoch nicht bei 100%.

Auf Github wurde zudem ein Workflow eingerichtet, welcher die Tests bei jedem Push ausführt.
